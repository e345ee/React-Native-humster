# StockExchangeRN

## Описание проекта

Я реализовал кроссплатформенное клиентское приложение на React Native и Expo для имитации торговой активности на бирже. Приложение подключается к внешнему backend API, получает рыночные котировки и данные портфеля, выполняет операции покупки и продажи активов, отображает статистику счёта и позволяет работать с приложением как на Android, так и в браузере.

Проект ориентирован на демонстрацию клиентской части торгового интерфейса. Основная бизнес-логика операций с пользователем, портфелем и торговыми запросами находится на стороне backend. Мобильное приложение выступает в роли frontend-клиента, который:

- авторизует и регистрирует пользователя;
- получает текущие котировки акций;
- отображает список инструментов и детали по выбранной акции;
- отправляет запросы на покупку и продажу;
- получает статистику портфеля и состояние баланса;
- позволяет пополнять счёт;
- поддерживает переключение светлой и тёмной темы.

## Функциональные возможности

### Аутентификация

Приложение поддерживает:

- вход по логину и паролю;
- регистрацию нового пользователя;
- сохранение авторизационного токена в локальном хранилище;
- восстановление сессии после перезапуска приложения.

### Работа с рынком

В приложении реализованы:

- загрузка котировок по набору тикеров;
- автоматическое обновление котировок на экране инструмента;
- отображение цены, абсолютного и процентного изменения;
- переход к карточке конкретной акции.

### Симуляция торговли

Приложение использует backend для торговых операций и отображает интерфейс симулятора биржевой торговли. Пользователь может:

- купить одну акцию по текущей цене;
- продать одну акцию по текущей цене;
- просматривать текущий объём позиции;
- видеть среднюю цену покупки, текущую стоимость позиции и финансовый результат.

Графики в карточке инструмента формируются на клиенте на основе текущей базовой цены и используются как визуальная симуляция динамики инструмента. Для отображения доступны линейный и свечной режимы.

### Портфель и статистика

Приложение отображает:

- денежный баланс счёта;
- количество сделок;
- агрегированную статистику покупок и продаж;
- общий объём вложений;
- чистый денежный поток;
- расчётную стоимость портфеля.

## Технологический стек

### Базовый стек

- **React Native** — кроссплатформенная разработка мобильного интерфейса;
- **Expo** — инфраструктура запуска, сборки и разработки;
- **TypeScript** — типизация приложения;
- **React 18** — базовая библиотека интерфейса.

### Навигация

- **@react-navigation/native**
- **@react-navigation/native-stack**
- **@react-navigation/bottom-tabs**

Используются стековая и табовая навигация.

### Работа с API

- **axios** — HTTP-клиент для взаимодействия с backend API.

В приложении настроен единый API-клиент с базовым URL и автоматической подстановкой Bearer-токена в заголовки запросов.

### Хранение состояния и локальные данные

- **zustand** — локальное состояние приложения;
- **@react-native-async-storage/async-storage** — хранение токена, пользовательских данных и кеша котировок.

### Поддержка интерфейса

- **react-native-safe-area-context** — работа с безопасными областями экрана;
- **react-native-gesture-handler** — поддержка жестов;
- **react-native-screens** — оптимизация экранов;
- **react-native-svg** — отрисовка графиков и SVG-элементов;
- **expo-status-bar** — управление статус-баром.

### Назначение каталогов

#### `App.tsx`

Корневая точка входа. Здесь инициализируются:

- контейнер навигации;
- провайдер безопасной области;
- статус-бар;
- тема приложения.

#### `src/api`

Слой взаимодействия с backend API.

- `client.ts` — экземпляр axios с базовым URL и токеном авторизации;
- `auth.ts` — запросы входа и регистрации;
- `market.ts` — получение рыночных котировок;
- `portfolio.ts` — операции с портфелем, балансом и торговыми сделками.

#### `src/screens`

Экранный слой приложения.

- `AuthScreen.tsx` — экран входа и регистрации;
- `StockListScreen.tsx` — список доступных акций;
- `StockDetailsScreen.tsx` — карточка акции, графики и торговые действия;
- `StatisticsScreen.tsx` — агрегированная статистика портфеля;
- `ProfileScreen.tsx` — профиль пользователя, баланс, настройки, выход.

#### `src/navigation`

Навигационная конфигурация приложения.

- `RootNavigator.tsx` — корневой стек, переключение между авторизацией и основной частью приложения;
- `MainTabs.tsx` — нижняя табовая навигация;
- `types.ts` — типы маршрутов.

#### `src/store`

Глобальное состояние приложения.

- `authStore.ts` — токен, данные пользователя, восстановление сессии;
- `themeStore.ts` — состояние темы интерфейса.

#### `src/components`

Переиспользуемые UI-компоненты:

- карточки;
- строки статистики;
- графики;
- контейнеры экранов;
- элементы списка инструментов;
- панели торговых действий;
- SVG-иконки.

#### `src/theme`

Описание цветовой схемы и получение текущей палитры интерфейса.

#### `src/types`

Типы DTO, ответы API и внутренние модели данных.

#### `src/utils`

Служебные функции:

- форматирование чисел и валют;
- подготовка данных для графиков;
- обработка ошибок API;
- уведомления.

#### `src/constants`

Константы приложения:

- список тикеров;
- валютные значения;
- конфигурация базового URL.

## Взаимодействие с backend

Базовый адрес backend задаётся через переменную окружения:

```env
EXPO_PUBLIC_API_URL=https://song-analysis.app/
```

По умолчанию проект использует удалённый API. При необходимости адрес backend можно переопределить локально.

### Пример файла `.env`

```env
EXPO_PUBLIC_API_URL=http://10.0.2.2:8080/
```

Для Android Emulator адрес `10.0.2.2` используется как ссылка на `localhost` хост-машины.

## API-методы, на которые опирается приложение

Приложение ожидает наличие backend-эндпоинтов следующего типа:

```text
auth/login
auth/register
market/quotes/:symbol
portfolio/statistics
portfolio/stocks/:symbol
portfolio/stocks/buy
portfolio/stocks/sell
portfolio/balance/top-up
```

Фактическая реализация и правила валидации этих эндпоинтов находятся на стороне backend.


## Запуск проекта

### Запуск в Android Emulator

1. Необходимо открыть Android Studio.
2. Необходимо запустить Android Emulator.
3. В корне проекта необходимо установить зависимости:

```bash
npm install
```

4. После этого необходимо запустить приложение:

```bash
npm run android
```

Данный сценарий запускает Expo Dev Server и открывает приложение в уже активном Android Emulator.

### Альтернативный Android-запуск через нативную сборку

Если требуется выполнить prebuild и собрать Android-проект в нативном режиме, необходимо использовать:

```bash
npm run native:android
```

Дополнительно доступны команды:

```bash
npm run prebuild:android
npm run prebuild
```

### Запуск в браузере

Проект может быть запущен как web-версия через Expo, поскольку в зависимостях присутствует поддержка `react-dom` и `react-native-web`.

Для запуска в браузере необходимо выполнить:

```bash
npm start
```

После запуска Metro Bundler в терминале необходимо выбрать открытие web-версии, например нажатием клавиши `w`, либо использовать соответствующий пункт интерфейса Expo.

Если требуется явно зафиксировать web-запуск отдельной командой, можно использовать:

```bash
npx expo start --web
```

## Логика пользовательского сценария

Типовой сценарий работы выглядит следующим образом:

1. Пользователь открывает приложение.
2. Пользователь проходит регистрацию или выполняет вход.
3. Приложение сохраняет токен доступа локально.
4. Пользователь открывает список инструментов и получает котировки.
5. Пользователь переходит в карточку акции.
6. Приложение показывает симулированный график и текущие данные по позиции.
7. Пользователь выполняет покупку или продажу, а приложение отправляет запрос на backend.
8. Пользователь открывает статистику и профиль для просмотра агрегированных результатов.

## Особенности реализации

- Тема интерфейса хранится локально и восстанавливается после перезапуска.
- Авторизационные данные сохраняются в `AsyncStorage`.
- Котировки частично кешируются локально для расчёта изменения цены.
- Графики на детальном экране генерируются на клиентской стороне как визуальная симуляция.
- Обновление котировки на экране инструмента выполняется автоматически с интервалом в 5 секунд.
